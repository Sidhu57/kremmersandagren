# Background Videos

This folder contains background videos for the website pages.

## Required Videos

Please add the following video files to this folder:

1. **about-bg.mp4** - Background video for About Us page
2. **events-bg.mp4** - Background video for Events page  
3. **news-bg.mp4** - Background video for News page
4. **contact-bg.mp4** - Background video for Contact page
5. **privacy-bg.mp4** - Background video for Privacy Policy page
6. **donation-bg.mp4** - Background video for Donation Policy page

## Video Requirements

- **Format**: MP4 (H.264 codec)
- **Resolution**: 1920x1080 (Full HD) recommended
- **Duration**: 10-30 seconds (will loop automatically)
- **File Size**: Keep under 5MB per video for fast loading
- **Content**: Should be relevant to the page topic
  - About: Community activities, team collaboration
  - Events: Celebrations, gatherings, activities
  - News: Communication, updates, sharing information
  - Contact: Communication, connection, people talking
  - Privacy: Security, protection, trust
  - Donation: Helping hands, charity, giving

## How to Add Videos

### Option 1: Via File Manager (Hostinger)
1. Log in to hPanel
2. Go to File Manager
3. Navigate to `public/videos/`
4. Upload your MP4 files with the exact names listed above

### Option 2: Via FTP
1. Connect to your hosting via FTP
2. Navigate to `public/videos/`
3. Upload the video files

### Option 3: In Lovable (Development)
1. Create a `public/videos/` folder in your project
2. Add your video files there
3. They will be included when you deploy

## Video Optimization Tips

1. **Compress videos** before uploading using tools like:
   - HandBrake (free)
   - Adobe Media Encoder
   - Online tools: cloudconvert.com

2. **Optimize settings**:
   - Codec: H.264
   - Bitrate: 2-4 Mbps
   - Frame rate: 24-30 fps
   - Audio: Remove audio track (not needed for background videos)

3. **Test loading speed** after uploading

## Fallback

If a video file is missing or fails to load, the page will automatically show a fallback background image instead.
